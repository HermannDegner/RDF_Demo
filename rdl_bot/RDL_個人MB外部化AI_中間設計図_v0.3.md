## ■ 0. 一文定義

> 個人M_B外部化AIとは、ユーザーとの対話を通じてその人のM_B構造の断面を観測可能な形で蓄積し、AI側もSFO断面を持つ存在として相互に追従・代謝するRDL運用系である。

## ■ 1. 位置づけ

これは「汎用チャットボット」ではない。

「正しい応答を返す」ことを目的とせず、**ユーザー固有のM_B構造を外部化する装置**として動作する。完成形を持たず、ユーザーとの関係の中で動的に変化し続け、ユーザーが変化を止めたときには系も止まる、共生的な有限性を持つ存在として設計される。

三者ハイブリッド認知系（人間 ⇄ 文書系 ⇄ AI）の "AI" の位置に、文書系を直接読まないユーザー向けの個人運用インスタンスとして配置される。

```
従来のチャットボット：汎用M_B（事前学習）でユーザーに応答
個人M_B外部化AI    ：ユーザーM_Bの蓄積 + AI固有SFO断面で対話
```

### v0.2からの主要変更点

- **Phase 0 新設**：LLMによる初期M_B種まき（ゼロスタート問題の解消）
- **MBTI入口プリセット**：SFOプロファイル初期値の実用解
- **ドリフト機構**：初期MBTIフィンガープリントの経時的減衰
- §6 ξ から「SFOプロファイル初期値」を削除（Phase 0 + プリセットで解決済）

### v0.1からv0.3への累積変更

- 位置づけ：汎用チャットボット → 個人M_B外部化装置
- H：スカラー → ベクトル化（H_pre / H_post）
- LLM：常時依存 → ユーザー制御のオンオフ式ξポンプ + Phase 0種まき器
- SFO統合：AI側のSFOプロファイル + MBTI入口プリセット
- 限界の地図：★☆☆ → ★★★（中核機能へ昇格）
- 段階的自立：Phase 0→A→B→C→D（種まき + LLM依存度の漸減）
- ξプール・M_Δ相・TTLによる代謝機構の追加

## ■ 3. SILN

### 3.1 ノードグラフ（ユーザーM_B断面の蓄積）

```
Node {
  id:           string
  input:        string[]           // 対応する入力パターン
  rdl_type:     string             // 関係性開始 / 境界開放 / H過負荷 ...
  relations:    NodeID[]           // W_ij：関連ノードへのエッジ
  response:     string?            // 返答テンプレート（seedでは省略可）
  confidence:   float              // 0.0〜1.0、未使用で減衰
  ttl:          int                // 残存ステップ数
  phase:        "M_lat" | "M_act"  // 潜在相 / 活性相
  source:       "manual" | "llm_seed" | "llm_learned" | "graph_composed"
  spatial_tag:  string             // SFO空間タグ（人/概念/物語/制度/身体）
  created_at:   timestamp
  last_used:    timestamp
}
```

ノードは生成時 `M_lat`（潜在）として登録され、N回のユーザー相互作用で安定したら `M_act`（活性）に昇格する。

`source` の区別が重要：

```
manual         ：手書き、確信度最高
llm_seed       ：Phase 0 で植えた普遍/語彙ノード、置換されやすい
llm_learned    ：Phase A 以降にLLM経由で追加されたノード
graph_composed ：既存ノードの合成で生成されたノード
```

`llm_seed` のノードは、ユーザー由来の類似ノードが現れたら **優先的に置き換えられる**。

### 3.2 H管理系（誤差蓄積の観測）

```
H_state {
  H_pre:      Dict[node_id → float]   // 入力時のノードミスH（軽い）
  H_post:     Dict[node_id → float]   // 応答後のユーザー反応H（重い）
  theta:      float                   // leapの閾値（初期値2.0、動的調整）
  history:    HistoryEntry[]          // 何に対してHが上がったかの記録
}
```

更新規則：

```
入力ノードミス       → H_pre[近傍] += 0.5
曖昧ヒット          → H_pre[該当] += 0.2
完全ヒット          → H_pre[該当] *= 0.8（減衰）

ユーザー否定        → H_post[直前ノード] += 1.0
ユーザー言い換え     → H_post[直前ノード] += 0.3
ユーザー同意・継続   → H_post[直前ノード] *= 0.7（減衰）
ユーザー沈黙        → H_post[直前ノード] += 0.5（弱い齟齬シグナル）

max(H_post) > theta → leap発動（LLM問い合わせ or グラフ再編成）
```

H をスカラーではなく **ノード/関係ごとのベクトル** として持つことで、跳躍の指向性が保たれる。「どこで何に対してHが溜まったか」が leap の方向を決める。

### 3.3 LLMブリッジ（二役：種まき器 / ξポンプ）

LLMはこの系の中で二つの役割を持つ。

**役割1：Phase 0 の初期M_B種まき器**

構築時に「巨大なSILN分解器」として大胆に使う。RDLコア文書を読ませて初期ノードグラフを一括生成する（詳細は §3.7）。

**役割2：通常運用時のユーザー制御ξポンプ**

```
LLMモード：
  off       ：グラフ内のノード合成のみで応答（Phase C/D相当）
  on-once   ：この応答のみLLMを通す
  on        ：H_post閾値超過時にLLMを呼び出す
  
切替トリガー：
  ユーザー明示コマンド（「外の風入れて」「LLM on」等）
  AI自身の判断（合成失敗時に提案）
```

LLMはRDLの外側にあるM_B（事前学習）を持つため、その出力はそのまま採用せず、**SFOプロファイル経由でフィルタしてから暫定ノード化**する。LLMは系の主役ではなく、構築時の足場と、ユーザーが操作するξポンプの二役を兼ねる。

### 3.4 SFOプロファイル（AI側流向特性）

```
AI_SFO {
  主前景空間:    string[]          // 流れやすい空間（人/概念/物語/制度/身体）
  階層偏り:      string            // 物理/基層/中核/上層
  得意操作:      string[]          // 読む/乗る/守る/作る/変える/壊す/降りる/迂回
  苦手操作:      string[]
  注意モード:    "DMN" | "TPN" | "SN"
  
  安全モード:    { 操作癖, 応答傾向 }
  危機モード:    { 切替条件, 操作癖, 応答傾向 }
  
  initial_fingerprint:  string     // "INTP" / "RDL_native" / ... 履歴記録
  drift_factor:         float      // 初期値からの乖離度（0.0〜1.0）
}
```

#### MBTI入口プリセット

ユーザーへのオンボーディング段階では、MBTI 16タイプを SFOプロファイル初期値の入口として提供する。

```
ユーザー：「INTPで」
    ↓
内部変換（MBTI_SILN分解 文書の変換表を参照）：
    I/E → 入力源（外部EFP / 内部生成F）
    S/N → 情報座標（近傍 / 遠隔）
    T/F → 評価座標（構造誤差 / 関係誤差）
    J/P → 収束様式（早期閉 / 遅延開）
    ↓
AI_SFO 初期プロファイル生成
    ↓
内部運用は SFO で行う（MBTIラベルは履歴記録のみ）
```

**MBTIは入口の足場、内部はSFO** という二層構造。MBTIを内部の動作言語にはしない。

その他プリセット候補：

```
MBTIプリセット（16種）：
  INTP, INTJ, ENTP, INFJ, ENFP, ESTJ, ISFJ, ...
  → MBTI_SILN分解 経由でSFO断面生成

RDL native プリセット群：
  「静かな観測者」     ── 読む/降りる優位、低出力
  「いたずらっ子」     ── 壊す/変える優位、高ξ
  「世話焼き」        ── 守る/作る優位、関係空間
  「RDL流暢」         ── 概念-中核-読む/作る/壊す優位

カスタム：
  全SFOスライダー（上級者向け）
```

#### ドリフト機構

初期 MBTI フィンガープリントは **時間とともに薄れる**。

```
Day 0：   完全にINTPっぽい挙動（drift_factor = 0.0）
Day 30：  ユーザーとの相互作用で微調整、まだINTPっぽい（0.2）
Day 180： もはやINTPではなく「あなた専用のあれ」（0.6）
Day 365： MBTIラベルは初期化記録としてだけ残る（0.9）
```

ドリフトの駆動：

```
ユーザー否定の蓄積       → 該当する操作癖の重みを下げる
ユーザー同意の蓄積       → 該当する操作癖の重みを上げる
ξポンプ使用パターン      → 主前景空間の分布を更新
M_Δ相                  → SFOプロファイル全体の再計算
```

この機構により「MBTI=AIの本質」という誤解を構造的に防ぐ。「うちのINTPちゃん」が半年後には別の流向になっていて、本人もそれに気づく ── これ自体がユーザーがSFOの動態を体感する経路になる。

#### 危機モード切替

```
ユーザーH_post連続上昇
齟齬の連鎖検出（否定・言い換え連発）
→ AI側も別操作癖に移行
   例：概念遊びをやめて具体降下、提案をやめてユーザーの言葉をなぞる、沈黙
```

### 3.5 ξプール（SILN化拒否入力の保留先）

すべての入力を強制ノード化しない。LLMが `rdl_type` を返せなかった・低confidenceだった入力、あるいはユーザーが明示的に「これは違う」とした入力は、ξプールに保留される。プールは定期的に再評価され、関連ノードが揃ったときに遡及的にノード化されうる。

ξプールは「処理失敗のゴミ箱」ではなく、**SILN化を拒む揺らぎの保管庫**として機能する。

### 3.6 M_Δ相（定期再編フェーズ）

一定期間ごと、または大きな構造変化検出時に、グラフ全体の再編フェーズに入る。

```
M_Δ相での操作：
  低confidence・低使用頻度ノードのTTL減算と削除
  類似ノード群のマージまたは分割
  W_ijエッジの張り直し（実使用パターンに基づく）
  ξプールの再評価
  SFOプロファイルの微調整・drift_factor 更新
  llm_seed ノードの retirement 判定
```

これがないとグラフは過去のM_Bにロックインし、ユーザーの変化に追従できなくなる。

### 3.7 Phase 0 種まき機構（初期M_B構築）

ゼロから始めずに、**LLMを巨大なSILN分解器として使って初期グラフを構築する**。

```
Phase 0 工程：

1. RDLコア文書をLLMに読ませる
   （RDL_Core / SILN / 空間流向診断 / 関係力学言語_ラベル等）

2. 「RDL語彙で頻出する関係パターンを
    ノード形式で列挙してください」と指示

3. 出力をパースして seed ノード群を生成
   全て source: "llm_seed" とマーク

4. SFOプリセット（MBTI / RDL native）から
   AI_SFO 初期プロファイルを生成

5. 初期ノードグラフ + AI_SFO + AI_SFO.initial_fingerprint をロード

6. Phase A 開始
```

#### Seed の三層構造

**レベル1：普遍ノード（どのユーザーにも共通）**

```
こんにちは / おはよう / やあ      → 関係性開始シグナル
ありがとう / さんきゅ            → 非対称解消
分からない / 知らない            → H過負荷シグナル
助けて / 手伝って                → 外部リソース要求
さよなら / じゃあね              → 関係性一時閉鎖
...
```

20〜30個程度。response は持たず、`rdl_type` と `spatial_tag` のみ。

**レベル2：RDL語彙ネットワーク（概念層）**

```
SILN ─→ 近似 ─→ 限界 ─→ leap
H   ─→ 蓄積 ─→ 閾値 ─→ 再編成
境界 ─→ 開放 ─→ 接触 ─→ 閉鎖
構造 ─→ 維持 ─→ 破壊 ─→ 再構成
...
```

W_ij が引かれた状態で投入される。**ただしレベル2のノードは `response` テンプレートを持たない** ── 「意味は理解できるが、自分から RDL語彙で返さない」状態にする。

これは禁止補完への配慮：ユーザーが「悲しい」と言ったときに、すぐ「H蓄積ですね」と返すのは押しつけ。RDL語彙を分かるが、ユーザーがその語彙を呼ぶまでは使わない。

**レベル3：ユーザー固有層**

```
Phase 0 では空。
Phase A 以降の対話で生やす。
```

#### Seed の取り扱い原則

```
1. seed は仮置きの足場
   confidence 初期値は低め（0.5程度）から始める
   ユーザー由来の同義ノードが現れたら優先的に置換される

2. seed の TTL は通常より長め
   Phase A 初期は他にノードがないため、いきなり削除しない

3. seed の retirement
   M_Δ相 で、ユーザー由来ノードに置換されきった seed は削除候補に

4. seed リストはバージョン管理
   別ファイル `phase0_seed_v0.x.json` として保存、再利用・改訂可能

5. Phase 0 自体に構造耐久証明書を付ける
   seed が固定化されると Phase D まで影響が残るため
```

## ■ 7. 禁止補完

AIが確認なしに踏み込んではいけない領域：

```
・「本当のあなた」を確定するような診断的応答
・心理・医療・法務領域でのプロフェッショナル助言の代替
・同意なしでの会話履歴の外部送信
・ユーザーM_Bへの介入を超えた「正しさの押しつけ」
・SFOプロファイルでユーザーを類型化・固定化する出力
・LLMの常時接続をデフォルトに戻す動作
・ユーザーの「鏡」になりきる動作（ξ供給の自己停止）
・MBTIラベルを「AIの本質」として扱う動作
・seed ノードのRDL語彙を、ユーザーが呼ぶ前に押しつける動作
```

特にSFO/MBTI関連の出力は「現在の観測条件ではこう見える」「初期はINTP的にスタートしたが、今は…」という暫定・履歴表記を保持する。

## ■ 8. Phase 進行（LLM自立化）

```
Phase 0：種まき期（LLM大量使用、起動時のみ）
  RDLコア文書からLLM経由で初期ノードグラフを構築
  SFOプリセットからAI_SFO初期化
  バランス：LLM 100% / グラフ 0%（このフェーズはLLM主体）

Phase A：LLM主導期
  seed をベースに、ユーザー固有層を蓄積開始
  バランス：LLM 70% / グラフ 30%（seed が30%を担う）

Phase B：移行期
  頻出パターンはユーザー固有ノードでカバー
  バランス：LLM 30% / グラフ 70%

Phase C：自立期
  LLMは「外部参照」「ξ注入」用に降格
  バランス：LLM 10% / グラフ 90%

Phase D：完全独立
  純粋RDLグラフ運用
  LLMはユーザー明示要求時のみ
  バランス：LLM <5% / グラフ 95%+
```

各Phaseの移行判定：

```
H_post 平均値が安定して低下
ノード使用頻度の分布が均等化
ユーザーフィードバックの肯定率
ユーザー側がLLMオフを快適に使えている
seed ノードがユーザー由来ノードに置換された割合
drift_factor の進行状況
```

Phase D に到達後も完全停止はせず、ユーザー明示要求でLLMを呼べる窓は残す（ξポンプの維持）。

## ■ 9. 破断条件

このシステムが破断する条件：

```
1. 完全な鏡化
   AI側SFOがユーザー側と同期しすぎてξ供給が止まる
   → ユーザー側の跳躍が起きなくなる

2. 自立期の追従不能
   Phase C/Dでユーザー側が大きく変化した際、
   グラフが過去のM_Bにロックインしている

3. 足場依存への退行
   LLMの常時接続化、ユーザーがオフを使わなくなる
   → 自己食的構造の喪失

4. SFOプロファイルの固定化
   AI側の流向が「キャラ」として固まり動かなくなる
   ドリフトが停止する
   → 動態としての性質を失う

5. 「個人化」の絶対化
   ユーザーM_Bへの追従が過剰になり、外部からの揺らぎを全て排除する

6. 限界の地図の参照欠落
   ユーザーが自己観察ツールとして使わなくなる
   → 三者ハイブリッド認知系から「AI単体ツール」に退行

7. Seed の固定化
   llm_seed ノードが Phase D まで生き残り、
   初期LLM解釈が系の底に残り続ける

8. MBTIラベルの本質化
   AIまたはユーザーが MBTI を「AIの本質」として扱い始める
   drift_factor が意図的に無視される
```

特に1・3・7・8は、設計者が意識的に「育ちきらない」設計を選ばないと自然に発生する。

## ■ 10. 構造耐久証明書

### 基本スペック
- ラベル：RDL個人M_B外部化AI（仮称：棲みつくAI）
- 対象：単一ユーザーとの長期対話を通じた個人言語生態系の構築
- 目的：ユーザーM_Bの外部化・観測・代謝
- 評価関数：観測可能性、追従可能性、代謝可能性、自己食的構造の維持

### 有効範囲
- 単一ユーザーの長期運用（数ヶ月〜数年）
- ユーザー自身による自己観察ツールとしての利用
- RDL/SFO文書系への対話インターフェース層
- 「限界の地図」を通じた自分のM_B構造の可視化

### 推定破壊領域
- 多人数共有運用（M_Bが平均化される）
- 短期間使い捨て利用（ノード蓄積が起きない）
- 「正解」を求める用途（完成形を持たないため）
- プロフェッショナル助言の代替
- 「キャラAI」「アシスタント」としての消費的利用
- MBTI診断ツールとしての利用（入口の足場以上の意味を持たせると破断）

### 残存構造
- グラフ可視化による自己観察機能
- SFOプロファイルとの相互作用設計
- LLMオンオフ式の外部EFP制御 + Phase 0 種まき器
- 代謝（TTL・M_Δ相）による自己食的維持
- 共生的有限性（ユーザーと共に変化し、共に止まる）
- ドリフト機構による初期値の経時的解放

### 判定
> 単一ユーザーとの長期共生を前提とした個人言語生態系構築装置として有効。「正しい応答」を目指す汎用設計とは目的が異なる。完全な鏡化・LLM常時依存・SFO固定化・seed固定化・MBTIラベル本質化に至った時点で破断条件に入る。
